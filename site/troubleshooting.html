Troubleshooting Guide

Solutions for common issues with the CLI Tool Context website

Step-by-Step Troubleshooting Guide

Follow these steps to diagnose and fix common issues with the CLI Tool Context website.

Step 1: Data Loading Checks

Check if data is loading properly:

  • Look for the tools count at the top of the page (should show "357+ tools")
  • If using file:// protocol, data loading may be restricted
  • Check browser console (F12) for network errors

Step 2: Console Error Checklist

Open browser console (F12) and check for:

  • Red error messages: JavaScript execution errors
  • Network failures: Failed to load JS/CSS files
  • Search worker errors: Web Worker initialization failures
  • CORS errors: Cross-origin request blocked

Step 3: File:// Protocol Guidance

If using file:// protocol (opening HTML directly):

  • Some browsers block local file access for security
  • Try a different browser (Chrome, Firefox, Safari)
  • Use a local web server instead (Python: python -m http.server)
  • Or use the live version at the GitHub Pages URL

Step 4: Search Worker Diagnostics

Test search functionality:

  • Try typing in the search box - results should appear
  • If search is slow or broken, check Worker support
  • Fallback search should activate automatically if Workers fail

Step 6: Browser Compatibility Info

Supported browsers:

  • Chrome/Edge: Version 60+ (recommended)
  • Firefox: Version 55+
  • Safari: Version 11+
  • Mobile browsers: Recent versions supported

Required features: JavaScript, Web Workers (optional), localStorage (optional)

Emergency Actions

Still Having Issues?

If problems persist:

  • Try incognito/private browsing mode
  • Clear browser cache and cookies for this site
  • Disable browser extensions temporarily
  • Check for browser updates
  • Report issues on GitHub
